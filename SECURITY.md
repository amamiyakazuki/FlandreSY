## 不要提交的内容

请确认这些文件不会进入 Git：

- `keystore.properties`
- `release-key.jks`
- `local.properties`
- 抓包文件，例如 `tmp_ujing_flows.jsonl`
- 任何包含 token、手机号、订单号、支付返回串的日志
- `.gradle/`、`build/`、`app/build/`
- `P_PLAN/`、`.codex/`、本地截图和调试过程文件

```powershell
git status --short

如果不确定某个文件能不能公开，就先不要提交。

## 日志

App 会写一些本地运行日志用于排查问题。日志里不应该保存完整支付返回串或 token。分享日志前仍建议手动看一眼，确认没有个人信息。

## 签名密钥

Release APK 的签名密钥代表这个 App 的身份。请把 keystore 和密码保存在本地安全位置，不要发给别人，也不要上传到 GitHub。

如果签名密钥泄露，别人可能可以构建看起来像同一个 App 的安装包。

## 漏洞反馈

如果你发现安全问题，请不要直接在公开 issue 里贴 token、手机号、订单号或抓包内容。可以先只描述问题类型和复现范围，再私下提供必要细节。

## 责任边界

本项目不是官方客户端。官方接口、支付流程、设备状态都可能变化。请不要把它用于绕过平台规则、批量请求、恶意测试或影响他人正常使用。
