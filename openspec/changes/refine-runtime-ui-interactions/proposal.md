## Why

当前 Compose 原型已经接入了一部分真实运行时能力，但页面上仍混有旧入口、静态假数据和错误跳转，导致用户很难判断哪些功能真实可用。现在需要把首页、订单、设备、我的页的交互边界重新整理：把输入项放到正确页面，把真实状态浮现出来，把无法验证的饮水能力明确降级，避免继续用洗衣链路假装饮水链路。

## What Changes

- 首页热水控制区移除手机号、密码、设备号等输入项，只保留紧凑的状态、开水、关水和错误提示；住理生活登录和热水设备码绑定移动到“我的”下的住理生活账户流程。
- 首页压缩为更接近参考图的静态比例，尽量在常见手机高度内展示完整主要内容。
- 首页“进行中”改为真实运行时任务区：无任务时显示“无任务”，有热水或洗衣任务时显示对应任务、计时/剩余时间，并支持点击进入相关详情页。
- 扫码结果必须分流：洗衣二维码进入洗衣下单，饮水二维码进入饮水相关页面或明确的待支持提示，未知二维码显示错误提示，不得默认跳到洗衣机界面。
- 订单页移除饮水和洗衣的静态假实现；洗衣订单使用真实订单状态，饮水订单在缺少已验证接口前显示“待接入/待捕获”状态，不伪造数据。
- 设备页改为可下拉刷新，刷新时/刷新后显示最近刷新时间；设备号单行省略，默认名称统一为“洗衣机A-01”这类短名称；编辑名称和删除设备变为真实本地设备列表操作。
- “我的”页账户卡片改为真实入口：住理生活与 U净都使用统一卡片结构；登录点击进入独立登录页；检测、重新登录、查看状态具备真实反馈；“解绑”改为“绑定设备码”，用于录入热水设备码并在状态页展示。
- 更多选项默认不展开，点击进入独立页面，承载权限检测、日志与诊断、导入/导出等功能入口，并为可实现项接入真实操作。
- **BREAKING**: 旧热水 Activity 不再作为用户可见入口保留；热水能力迁移到 Compose 运行时和“我的”配置流程，遗留 Activity 可删除或仅作为内部兼容代码逐步清理。

## Capabilities

### New Capabilities
- `runtime-ui-interactions`: 覆盖首页真实任务展示、扫码分流、订单/设备/我的页真实交互、账户配置页和更多选项页的运行时 UI 行为。

### Modified Capabilities
- `runtime-order-continuity`: 修改旧热水入口策略，并把运行中任务恢复、订单详情、支付后启动边界与首页任务展示保持一致。

## Impact

- Compose UI: `ShuiScreens.kt`、`ShuiComponents.kt`、导航状态、首页/订单/设备/我的页面组件。
- Runtime/ViewModel: `ShuiRuntime.kt`、热水与 U净 adapter 的状态暴露、扫码结果识别、任务状态聚合。
- Hot water: 住理生活登录、设备码持久化、开关水状态、旧 Activity 入口清理。
- Ujing washer: 洗衣扫码、设备列表、订单详情、运行中订单、支付/启动/停止链路。
- Storage: SharedPreferences 中账号、设备码、设备别名、本地设备列表和最近刷新时间。
- Documentation/specs: 饮水接口缺少抓包依据，必须标注为待接入，不得使用洗衣接口伪实现。
